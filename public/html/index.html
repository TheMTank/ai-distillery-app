Vision

Our plan, our mission and our goals. You can read about MTank's vision here.

Modeling AI Knowledge

Handling AI knowledge is becoming increasingly difficult due to the large amount of paper published on open access platforms like arXiv.org. Effectively modeling data is necessary in order to allow people to access and explore information freely.

Automatically Extracting Knowledge from Data

Due to the large amount of data, knowledge extraction cannot be a manual process: we employ automatic techniques to extract information from papers and use this to analyze them.

Sharing Knowledge with the Community

We extract knowledge and we give free access to it for the community. Everybody should be able to access to this data and to use it. Even a novice with little understanding of AI should be given a place to see easily where to AI knowledge progressing.

Paper Search

Search for papers with free-text. Just input a keyword and get the top matched papers. Similar in functionality to arxiv-sanity.

Word Embedding Proximity

Search for similar words using a any word found within the corpus as input (typeahead helps). Search is done in a vector space where words are represented by vectors. We currently use two embedding methods: Word2Vec and FastText.

Check out our Word Embeddings within and compare between the types

Paper Embedding Proximity

Search for similar embeddings to the searched paper (with help from a typeahead and random selection to help you choose). No room for misspellings (but case doesn't matter), paper title has to be exact, hence the typeahead is necessary.

Word Embedding Visualisation

Explore through the TSNE reduced embedding space of FastText or Word2Vec

The left side allows the user to query a specific embedding by it's label, the number of vectors, number of clusters and embedding type.
The right side side allows the user to choose specific papers and see the cluster centroids of K-means.

Paper Embedding Visualisation

Search the T-SNE space of paper embeddings and find that similar papers lie close in this space. You can explore these spaces created from two different methods: Doc2Vec and Latent Semantic Analysis (LSA)

The left column within the tool allows the user to query a specific embedding by it's label, the number of vectors, number of clusters and embedding type.
The right side side allows the user to choose specific embeddings and see the cluster centroids of K-means.
